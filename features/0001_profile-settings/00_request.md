# Feature Request: Profile Settings Page

Users need a place to manage their account. Right now there is no settings page at all.

## What the page should let users do

**Display name**
Users can update their display name. This is the name shown throughout the app.

**Email (read-only)**
Show the user's current email address but do not allow editing it. No email change flow for now.

**Password**
Users can change their password. They must enter their current password first, then the new password twice to confirm.

**Delete account**
Users can permanently delete their account. This is destructive and irreversible, so it needs a confirmation step — for example a modal asking them to confirm before proceeding. Deleting an account should remove all user data.

## What is out of scope for this version

- Avatar / profile photo
- Email change
- Notification preferences
- Learning preferences
